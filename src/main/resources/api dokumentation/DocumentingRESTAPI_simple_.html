Request Body and Response Formats

(1) Address format (don’t provide ID, for POST and PUT. PUT only need to contain data that has to be updated)

 {
   "street": String,
   "additionalInfo": String,
   “innerCityInfoDTO”:

{

         “id”: int,

         “zipcode”: String,

         “city”: String,

}


 }

(2) Cityinfo format (don’t provide ID, for POST and PUT. PUT only need to contain data that has to be updated)

 {
   "zipcode": String,
   "city": String,
 }

(3) Hobby format (don’t provide ID, for POST and PUT. PUT only need to contain data that has to be updated. amountOfHobbyists cant be set)

 {
   "name": String,
   "description": String,

    “amountOfHobbyists: int,
 }

(4) Person format (don’t provide ID, for POST and PUT. PUT only need to contain data that has to be updated)

 {
   "email": String,
   "firstName": String,

    "lastName": String,
   "address":

    {
   “innerCityInfoDTO”:

{

         “id”: int,

         “zipcode”: String,

         “city”: String,

}

    },

    "hobbies": [

        {

         “id”: int,

         “name”: String,

         “description”: String

}

    ],

    “phones”: [

         {

         “number”: int,

         “description”: String

}

    ],

           }
}

(5) Person Hobby Amount format

 {

        “amountOfHobbyist”: int

 }

(6) Phone format (don’t provide ID, for POST and PUT. PUT only need to contain data that has to be updated)

 {
   "number": int,
   "description": String,
   “person”:

{

         “id”: int,

         “email”: String,

         “firstName”: String,

         “lastName”: String,

}


 }

Errors

(e) All errors are reported using this format (with the HTTP-status code matching the number)

{ status : statusCode, "msg": "Explains the problem" }

  • (e1) :{ status : 404, "msg": "Could not find ‘xxx’ with id:" } (for example, wrong id)
